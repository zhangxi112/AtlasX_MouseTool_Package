# Atlas-X Cursor Studio 鼠标指针自定义工具软件 V1.0 软件设计说明书

## 1 软件设计目标

本软件目标是在 Windows 桌面环境中提供可运行的鼠标指针自定义工具，使用户能够选择内置指针主题、导入图片生成自定义光标、应用到系统光标、恢复系统默认设置，并通过找回鼠标、晃动尾迹、点击波纹和按程序切换等功能提升鼠标可见性。软件正式产品名称为 Atlas-X Cursor Studio。软著登记全称为“Atlas-X Cursor Studio 鼠标指针自定义工具软件 V1.0”，其中“鼠标指针自定义工具软件”为功能描述，“V1.0”为登记版本号。软件不包含账号系统、云同步、服务端数据库或会员功能。

## 2 总体架构

软件采用 Python + PySide6 的本地桌面架构。本软件使用 PySide6 Community Edition，并按 LGPLv3 路径履行许可义务。本软件未选择 PySide6 的 GPL 授权路径，项目自身代码不因使用 PySide6 而改用 GPL。本项目未修改 PySide6、Qt、Shiboken 源码。PySide6、Qt、Shiboken 及其相关组件不作为张希的原创源码提交。PySide6 用于 Qt 桌面界面、窗口控件、托盘和覆盖层窗口。第三方依赖仅作为运行依赖或构建依赖使用，不纳入软著原创源程序鉴别材料。`app/main.py` 创建 Qt 应用、主窗口、托盘热键和异常钩子；`app/bootstrap.py` 负责加载配置并装配服务对象。`core` 提供常量、数据模型和内置主题定义。`services` 承担光标管理、配置、图片处理、覆盖层显示、运行时增强、前台窗口检测、开机启动、日志和卸载管理。`ui` 负责主窗口、各页面控件和用户交互。构建阶段由 PyInstaller 和 Inno Setup 生成可执行程序、便携包和安装器。

## 3 模块划分

- `app`：程序入口、应用上下文、异常恢复和生命周期管理。
- `core`：软件名称、版本、数据结构、内置主题元数据。
- `services/config_manager.py`：默认配置和用户配置合并、保存、运行目录解析。
- `services/cursor_manager.py`：内置主题渲染、`.cur` 文件生成、系统光标应用和恢复。
- `services/image_pipeline.py`：图片导入、抠图、裁剪、缩放、预览和自定义光标生成。
- `services/overlay_manager.py`：透明顶层覆盖窗口，用于找回鼠标、尾迹和点击波纹。
- `services/enhancement_manager.py`：动态光标、按程序切换、点击与移动轮询、全屏暂停协调。
- `services/tray_hotkey_manager.py`：系统托盘和全局快捷键。
- `services/startup_manager.py`：当前用户开机启动注册表项。
- `ui/main_window.py`：主界面和设置交互。

## 4 数据流程

启动时，程序读取 `configs/default_config.json`，再读取用户目录中的 `config.json` 并合并为运行配置。用户选择内置主题后，界面把主题 ID、配色和大小传给光标管理服务，服务生成或复用 `.cur` 文件并调用系统接口应用。用户导入图片后，图片处理服务把源图复制为工作 PNG，按用户设置生成预览；用户确认后生成 `.cur` 文件，光标管理服务应用到系统，配置管理服务保存当前主题和最近记录。运行过程中，增强管理服务轮询前台窗口、鼠标点击和移动状态，必要时触发覆盖层显示或切换主题。

## 5 主要类、函数或模块说明

- `AppContext`：保存配置、日志、光标、覆盖层、增强、图片处理、启动项和卸载服务实例。
- `ConfigManager.load/save`：读取、合并、校验并保存配置。
- `CursorManager.apply_cursor_file`：通过 Windows API 将指定 `.cur` 文件应用到系统光标角色。
- `CursorManager.restore_system_default`：调用系统参数接口恢复 Windows 默认光标。
- `ImagePipeline.import_image/remove_background/generate_preview/generate_cursor`：完成自定义图片到光标文件的处理流程。
- `EnhancementManager.refresh_from_config`：根据配置启动或停止前台程序轮询、点击轮询、移动轮询和动画定时器。
- `OverlayManager.trigger_find_mouse/trigger_motion_trail/trigger_click_ripple`：显示桌面级提示效果。
- `StartupManager.set_enabled/repair_if_needed`：写入或修复当前用户开机启动命令。

## 6 配置文件设计

默认配置位于 `configs/default_config.json`，字段包括 `version`、`app_state`、`appearance`、`highlight`、`pointer`、`click_ripple`、`dynamic_cursor`、`app_switch`、`game_mode` 和 `startup`。用户配置优先写入 `%APPDATA%/atlas_x_cursor_studio/config.json`，无法创建用户目录时回退到 `configs/runtime`。配置保存使用 UTF-8-SIG JSON，便于中文字段和 Windows 环境读取。

## 7 图标资源处理流程

内置主题由 `core/builtin_themes.py` 定义主题 ID、名称、说明、热点坐标和默认色板。`assets/builtin_cursors` 下 png 和 cur 文件按项目生成的内置鼠标指针资产处理。`CursorManager` 使用 Pillow 绘制箭头、圆点、十字、闪电、圆环、定位针等图形，生成预览 PNG 和 Windows `.cur` 文件。`assets/app_icon.ico` 由本项目生成，作为项目图标资产处理。自定义图片流程为：校验格式、转换 RGBA、可选本地 U2Net 抠图、裁剪透明边缘、缩放居中、设置热点、生成预览、生成 `.cur`、应用系统光标。

## 8 系统接口或权限调用说明

系统光标应用使用 Windows `user32.dll` 中的 `LoadImageW` 和 `SetSystemCursor`，恢复默认光标使用 `SystemParametersInfoW(SPI_SETCURSORS)`。开机启动使用 `HKCU\Software\Microsoft\Windows\CurrentVersion\Run\AtlasXCursorStudio`，属于当前用户注册表项。全局快捷键和鼠标状态检测使用 Windows 用户态接口。软件未发现驱动安装、服务端通信或管理员强制提权设计。

## 9 异常处理设计

服务层对不存在的文件、不支持的图片格式、Windows API 调用失败、注册表写入失败、快捷键解析失败和抠图依赖缺失进行捕获或返回失败状态。界面层通过警告框、信息框和状态栏向用户反馈。主入口安装未处理异常钩子，异常发生时写日志并尝试恢复系统默认光标。

## 10 打包与发布流程

`atlas_cursor_studio.spec` 指定 PyInstaller 入口为 `app/main.py`，收集 `configs/default_config.json` 和 `assets`，生成 `AtlasXCursorStudio` 可执行目录。`scripts/build.ps1` 执行基础构建，`scripts/build_release.ps1` 生成便携 zip，并在检测到 Inno Setup 6 时调用 `scripts/installer.iss` 生成 `AtlasXCursorStudio-Setup-1.0.exe`。发布产物位于 `dist` 和 `release`，不纳入原创源码材料。

## 11 安全与隐私说明

软件配置、日志和用户导入的自定义光标工作文件保存在本地用户目录。源码未发现账号登录、云同步、数据库后台或主动上传用户文件的功能。自动抠图依赖本地 ONNX 推理，README 提到首次使用自动抠图可能联网下载 `u2net.onnx` 模型，模型下载来源和授权需人工确认。
